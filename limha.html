Overview

Limha is a self-hosted home automation hub, and the project on the bio page that started with wanting to write some code™ in my spare time instead of just installing Home Assistant. It talks to real hardware and real protocols: HomeKit, MQTT, LG ThinQ, Sonos, Shelly, UniFi Protect cameras, and a scattering of Wemos D1 mini sensors running a custom Arduino sketch.

It started life in 2020 as limha_rust, a first attempt written in Rust, before getting rewritten in Elixir and Phoenix in 2021 — which is what it's been ever since. A dashboard builder lets me lay out drag-and-drop views per room, and a visual workflow editor handles automations (states, conditions, actions) instead of a background cron job quietly doing things I can't see.

One of the more fun rabbit holes: getting live UniFi Protect camera video playing in the browser without RTSP or an external transcoder. Protect exposes a ticketed livestream WebSocket that isn't documented anywhere — each message is framed with a type/length header, and a playable video fragment has to be assembled across several of those frames before it can be handed to the browser's Media Source Extensions. Getting that framing right took a few tries, but it means the camera feed plays natively with zero extra infrastructure.

These days Limha runs in production on a k3s cluster in a server rack in my house, alongside Nimbus, which is what actually provisions and runs that cluster.
