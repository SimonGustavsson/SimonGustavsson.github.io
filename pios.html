PiOS

A bare-metal operating system for the Raspberry Pi, written from scratch in C and ARM assembly.

[Screenshot placeholder]

Overview

PiOS started the way a lot of these things do: the Raspberry Pi 1 came out, I wanted to know exactly how a computer boots without an OS underneath it, and a blinking LED driven by a few lines of raw ARM assembly was the first thing I got working. From there it kept growing.

By the time I moved on, PiOS could talk to GPIO, draw pixels to a framebuffer, render a character-based terminal, handle interrupts and a system timer, read and write eMMC storage, and walk a (read-only) FAT32 filesystem. It had a working MMU with proper virtual memory — high memory for the kernel, low memory for user code — could load and execute ELF binaries, talk over UART, and print a stack trace when something faulted, which by that point in a bare-metal OS was less "nice to have" and more "the only way to debug anything at all."

No libraries, no existing OS underneath it to lean on — just the datasheet and the metal. It ran from 2013 to 2014, about 446 commits, and taught me more about how a computer actually works than anything before or since.
